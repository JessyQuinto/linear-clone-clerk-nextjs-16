# Plan: Complete Project Administration вҖ” Clean Linear-style Interface

## Problem

The project detail page (`/projects/[projectId]`) has 4 tabs, but 3 of them use hardcoded mock data:
- **Iniciativas** вҖ” shows hardcoded initiative descriptions and fake progress values (though it queries `api.cycles.listWithProgress`)
- **Soluciones** вҖ” 100% hardcoded `solutionsData` array (RAG Motor, Cotizador, etc.)
- **Testing & QA** вҖ” 100% hardcoded `qaData` array

Only the **Slices/Tareas** tab uses real backend data. The cycles page also has a hardcoded SVG tree visualization. The interface needs to be cleaned up to match Linear's minimal, data-driven aesthetic.

## Approach

Strip the project detail page down to a clean Linear-style layout:
- Header with breadcrumb + "Add issues" button
- Editable name + description
- Progress bar with real issue counts
- **Primary content: real issues list** (grouped by status) вҖ” this IS the main view, not a tab
- **Sidebar: real project properties** (status, lead, date, color, delete)
- **Add "Cycles" section** below issues if cycles are relevant (linked cycles for the project's team)
- Remove all tabs, all mock data, all hardcoded solutions/QA
